Language models learn a probability distribution over sequences of tokens. A decoder predicts the next token from the tokens that came before it. Attention lets each position combine information from earlier positions. A causal mask prevents a token from observing the future. Training repeatedly compares predicted logits with the expected next token and updates the parameters. Small experiments are useful for checking a data pipeline before expensive training begins.

Reliable machine learning projects separate source code, configuration, data, checkpoints, and tests. Configuration files record the choices used for an experiment. Processed data should be reproducible from documented raw inputs. Checkpoints should contain enough metadata to resume a run safely. Tests should use tiny models so they can run without a GPU. Clear boundaries make experiments easier to review, reproduce, and extend.

Vision-language models connect an image encoder to a language model through a learned projection. The image encoder produces patch features. The projection maps those features into the token embedding space. During alignment training the language model can remain frozen. During instruction tuning selected language parameters can be updated so that the model learns how to answer questions about images.
